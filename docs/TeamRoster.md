# Team Roster & Repository Tracks

This workspace manages two distinct development tracks: **Core** (Professional/General) and **Personal** (Religious).

## 1. Core Track (General/Work)
General-purpose power tools designed for any high-reasoning task. Master copies reside in `src/skills/core/` and `src/tools/core/`.

| Role | Skill/Tool | Trigger / Use Case |
| :--- | :--- | :--- |
| **Architect** | `project-planner` | Translates a vague idea into a high-level `Plan_*.md`. |
| **Tactical Architect** | `coding-step-planner` | Breaks a single step from a Plan into a `Spec_*.md`. |
| **Lead Researcher** | `autonomous-researcher` | Deep-dives into unknown libraries, APIs, or documentation. |
| **Forensic Investigator** | `code-debugger` | Automatic troubleshooting triggered by `[DIAGNOSTIC MODE ACTIVE]`. |
| **Security Officer** | `skill-auditor` | Automated security and integrity scanner. |
| **Web Explorer** | `interactive-web-explorer` | Playwright-based interactive browsing. |
| **Home Manager** | `home-assistant` | Controls the local smart home via MCP. |
| **Workspace Manager** | `google-workspace-mcp` | Full CRUD for Gmail, Tasks, Calendar, and Drive via MCP. |

## 2. Personal Track (Religious/Custom)
Specialized tools built on Core foundations for specific workflows. Master copies reside in `src/skills/personal/` and `src/tools/personal/`.

| Role | Skill/Tool | Trigger / Use Case |
| :--- | :--- | :--- |
| **JW Assistant** | `jw-assistant-toolkit` | Grounding instructions for religious research. |
| **Orchestrator** | `jw-daily-orchestrator` | Automation for Daily Text synthesis. |
